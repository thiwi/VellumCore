# Security Policy

## Supported versions

Vellum Core follows SemVer. The latest major and latest minor releases are
prioritized for security fixes.

## Reporting a vulnerability

Please do not report security vulnerabilities in public issues.

- Send details privately to project maintainers.
- Include reproduction steps, impacted components, and severity estimate.
- If possible, include a minimal proof of concept.

## Response targets

- Initial acknowledgement: within 3 business days.
- Triage and severity assessment: within 7 business days.
- Mitigation plan and patch timeline: communicated after triage.

Operational hardening guidance remains documented in `docs/SECURITY.md`.
